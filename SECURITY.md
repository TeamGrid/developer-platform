# Security policy

## Supported versions

Only the newest published prerelease or stable version of each package is
supported. Alpha releases may change before the stable `1.0.0` release.

## Reporting a vulnerability

Do not open a public issue for a suspected vulnerability. Use GitHub's private
vulnerability reporting for this repository. Include the affected package and
version, impact, reproduction steps, and any suggested mitigation.

Never include TeamGrid API credentials, customer data, webhook signing
secrets, or other production secrets in a report or test case.
